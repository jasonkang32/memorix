# Memorix Secret Storage Design

## 목표

- `Work`를 기본 영역으로 둔다.
- 기존 `Personal` 개념은 `Secret`으로 통합한다.
- `Secret`은 패스워드 또는 생체인증으로만 접근한다.
- `Secret`에 저장된 미디어는 일반 파일 탐색기나 갤러리에서 바로 열리지 않도록 한다.
- 사용자가 원본 삭제 여부를 직접 선택할 수 있게 한다.
- 앱 삭제 후에도 보관 데이터가 유지되도록 별도 백업/복원 경로를 둔다.

## 핵심 원칙

1. 원본 미디어와 앱 보관함 미디어를 분리한다.
2. 보관함 데이터는 암호화한다.
3. 목록 화면은 가벼운 썸네일만 먼저 읽는다.
4. 상세 화면에서만 원본을 복호화한다.
5. 앱 삭제 후 유지가 필요한 데이터는 앱 전용 내부 저장소에만 의존하지 않는다.

## 권장 구조

### 1. 앱 영역

- `Work`
  - 업무용 기본 공간
  - 빠른 등록과 조회에 집중
- `Secret`
  - 잠금이 필요한 통합 보관 영역
  - 개인 사진과 숨기고 싶은 업무 사진을 함께 관리

### 2. 저장 계층

- `media metadata`
  - DB 저장
  - 예: id, space, label, thumbnail key, encrypted file key, size, hash, createdAt
- `thumbnail cache`
  - 작은 미리보기만 저장
  - 목록 화면에서 우선 사용
- `encrypted original vault`
  - 원본 이미지는 암호화 파일로 저장
  - 일반 파일명, 일반 확장자 노출 최소화

## 저장 방식

### 비추천

- 원본 이미지를 SQLite `blob`에 그대로 저장

이 방식은 다음 문제가 있다.

- 큰 사진이 많아질수록 DB가 비대해진다.
- 목록 로딩과 백업/복원 비용이 커진다.
- 검색, 마이그레이션, 동기화 시 부담이 늘어난다.
- 여러 장 동시 등록 시 체감 성능이 떨어질 수 있다.

### 권장

- 원본은 앱 전용 암호화 파일로 저장
- DB에는 메타데이터만 저장
- 썸네일은 별도 경량 캐시로 저장

이 방식의 장점:

- 목록 로딩이 빠르다.
- 파일 단위 관리가 쉽다.
- 암호화 적용이 단순하다.
- 향후 클라우드 백업으로 옮기기 쉽다.

## 삭제 UX

### 저장 후 흐름

1. 사용자가 사진/영상을 선택한다.
2. 앱이 보관함에 복사하고 암호화한다.
3. 상세 입력을 마치고 저장한다.
4. 저장 완료 후 원본 삭제 여부를 묻는다.
5. 사용자가 동의하면 원본 파일을 삭제한다.

### 권장 문구

- "메모릭스 보관함에 복사되었습니다."
- "갤러리에 원본을 그대로 두면 같은 파일이 두 곳에 남습니다."
- "메모릭스에만 보관하려면 원본 삭제를 권장합니다."

### 삭제 정책

- 기본값은 삭제하지 않음
- 사용자가 명시적으로 동의해야 삭제
- 삭제 실패 시 이유를 간단히 보여줌

## 잠금 정책

`Secret` 진입 시 다음 순서로 인증한다.

1. 생체인증 시도
2. 실패 또는 미지원이면 앱 비밀번호 입력
3. 일정 시간 비활성 상태면 재잠금
4. 앱 전환 시 최근 화면 미리보기 최소화

## 앱 삭제 후 유지

로컬 내부 저장소만으로는 앱 삭제 후 데이터를 남길 수 없다. 따라서 아래 두 단계로 나눈다.

### 1. 기본 보관

- 암호화된 내부 vault에 저장
- 탐색기와 갤러리 노출 방지

### 2. 장기 보존

- 사용자가 선택하면 암호화 백업을 외부 저장소 또는 클라우드로 내보냄
- 앱 재설치 후 복구키 또는 비밀번호로 복원

이 구조가 현실적이다.

- 보안성과 은닉성을 유지할 수 있다.
- 앱 삭제 후 보존 요구를 만족시킬 수 있다.
- 운영체제 삭제 정책에 덜 흔들린다.

## 구현 우선순위

1. `Work` / `Secret` 모델 정리
2. `Secret` 잠금 진입 추가
3. 저장 경로를 암호화 vault로 분리
4. 썸네일 캐시 도입
5. 저장 후 원본 삭제 선택 UI 추가
6. 암호화 백업/복원 설계 추가
7. 앱 삭제 후 복원 흐름 문서화

## 현재 결론

- `blob` 원본 저장은 권장하지 않는다.
- `Secret`은 별도 잠금 보관함으로 통합하는 편이 맞다.
- 앱 삭제 후 유지가 필요하면 암호화 백업을 별도 기능으로 둬야 한다.
- 화면 구조는 `Work` 중심, `Secret` 잠금 진입 구조가 가장 단순하다.
